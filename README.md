ü¶à Wireshark Analysis

Wireshark was used to capture and analyze the network traffic generated by the Nmap SYN scan, confirming the underlying TCP handshake behavior.

1. SYN Packets (Nmap Scan Requests)

Filter Used: tcp.flags.syn == 1 and tcp.flags.ack == 0

Purpose: This filter displays packets where the SYN flag is set but the ACK flag is not, which corresponds to Nmap sending its initial probe packets to various ports.

Visual Evidence: wireshark_syn.png

2. SYN-ACK Packets (Open Ports Responding)

Filter Used: tcp.flags.syn == 1 and tcp.flags.ack == 1

Purpose: These are the responses from active hosts that have an open port. The host responds by setting both the SYN and ACK flags, indicating the port is ready to establish a connection.

Visual Evidence: wireshark_synack.png

3. RST Packets (Closed Ports Responding)

Filter Used: tcp.flags.reset == 1

Purpose: These packets indicate that a closed port rejected the connection attempt from Nmap by sending a Reset (RST) flag.

Visual Evidence: wireshark_rst.png
